# TODO: 支持发送文件

## 目标

在 TIMBot 中支持“将长内容导出为文件并发送”，作为超长回复的替代承载方式。

## 背景

当前长文本超限后的默认方案是：

- 停止继续往同一条流式消息里追加
- 将最终文本按固定长度拆分
- 剩余内容继续以多条文本消息发送

这能解决“消息体过长直接失败”的问题，但不适合以下内容：

- 超长代码
- 超长 Markdown 文档
- 长表格
- 教程/报告类输出

这些内容更适合直接转成文件发送。

## 预期形态

理想流程：

1. 流式预览到安全上限附近停止
2. 生成完整文件
3. 聊天内发送一条短提示
4. 发送文件，或发送文件入口/下载方式

## 优先级

建议优先级：高

原因：

- 这是比“继续拆消息”更自然的超长内容承载方式
- 对代码、长文、表格类回答更友好
- 能减少 Markdown 被中间切开的显示问题
