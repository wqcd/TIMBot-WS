# TODO: 默认启用流式输出

## 目标

在 TIMBot 中将回复模式的默认值从 `off` 改为某种流式模式（优先考虑 `tim_stream`），让大部分用户在无需额外配置的情况下直接享受流式体验。

## 背景

当前 `streamingMode` 的默认值是 `off`，即：

- 不走流式
- 等 agent 生成完毕后一次性发送完整回复

原因主要是历史上客户端对流式能力的支持不一致（尤其是旧版本、多端登录场景），为了兼容性暂时选择保守策略。

随着：

- 新版客户端对 `TIMStreamElem` 的支持逐步完善
- `CompatibleText` 能在旧客户端做兜底
- 业务侧对「类 ChatGPT 流式体验」的需求增强

可以考虑把默认体验从“非流式”升级为“流式优先”。

## 预期形态

理想流程（以 `tim_stream` 为默认为例）：

1. 用户发送提问
2. TIMBot 创建一条 `TIMStreamElem` 流式消息
3. 新客户端实时渲染 chunk，旧客户端看到 `CompatibleText` 的合理兜底内容
4. 流式完成时：
   - `IsLast = true` 的最终 chunk 发送完毕
   - 必要时再补一条短提示或摘要
5. 如遇故障，自动按策略降级到普通文本消息，保证用户始终能看到最终回答。

## 优先级

建议优先级：中-高

原因：

- 直接影响用户对「像 ChatGPT 一样流式思考」的感知体验
- 在客户端能力成熟后，把默认从“非流式”切到“流式”是自然演进
- 配合其他体验优化（如发送文件）能构成更完整的长对话体验
